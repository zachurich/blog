Different Title

It’s easy to get lost in all the resources made available to us when learning to code. From completing tutorials on Codecademy, reading dev articles on Medium, and going through tracks on Freecodecamp, it can be quite overwhelming.

The overwhelming nature, at least for me, came from fracturing my attention between these resources and many more, all sending ideas my way that often followed different patterns and teaching methods that left me feeling uneasy about my skills.

The Problem

I’d read an article on medium where the author would discuss some upcoming CSS feature, another about the best Javascript frameworks (yikes), and then go and complete a tutorial that yet another article pointed to as a resource on a completely separate topic. As I continued on this path, I noticed confidence in my own skills began to dwindle. Why?

Every resource threw some new development concept at me, keeping me in a state of feeling like I wasn’t learning quick enough or retaining everything I should be.

I believe this issue, which is something I still struggle with, comes from untamed curiosity. I want to learn how the web works, and how to create usable products for it, so I try to learn as much as I can too quickly, feeding that cycle of feeling inadequate.

This cycle is terrible for anyone trying to learn a new skill, as it can feed Impostor Syndrome. While Impostor Syndrome is an important factor here, I won’t discuss it as there are many articles and studies covering the topic at length, specifically pertaining to development and design.

How to Break the Cycle

Breaking this overwhelming cycle has many benefits, but it can be done through focus and and building good habits. For one, scrap the whole idea of scouring the web and completing tutorials in hopes of cramming knowledge and becoming an expert on everything quickly.

I will say, this “shotgunning” has its place for complete beginners when building associations and high-level concepts, but if you are reading this, I’m willing to bet you are a little further along in your learning journey. Start focusing in on one thing you want to learn about, and create a project focusing on that one idea.

The project doesn’t need to be extensive or elaborate, but it should focus on a specific function built upon a single concept you want to learn. I will say, this method can get boring if your idea doesn’t incorporate something that interests you or that you find valuable, so make sure you make something that you personally find fulfillment in creating.

If you are on a journey to learn how to make cool things for the web, recognize that absorbing everything will take time. Instead of continually shotgunning knowledge into your mind, set aside time to pursue one development concept, and build something cool around that. You will be surprised as to how much more you will retain because you will be building something you personally appreciate.
